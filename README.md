# Data Desensitization Skill

一个面向 AI 协作场景的通用数据脱敏 skill。

它不是“简单打码工具”，而是一套可复用的脱敏方法：  
**识别敏感字段、泛化业务标识、扰动全部数字、重写对外表达，并尽量保留分析结构。**

适合你把真实业务数据安全地变成：

- 可外发的 Excel / CSV
- 可分享的 PPT / Word / HTML
- 可复用的案例材料
- 可交给 AI 继续处理的脱敏内容

---

## 为什么这个项目有价值

很多团队做“脱敏”时，只处理姓名和手机号，但真正会泄漏业务信息的，往往还有：

- 品牌名、门店名、策略名、内部渠道名
- 会员 ID、用户 ID、订单号
- 人数、金额、GMV、ROI、客单价、订单量、转化率
- 图表标签、tooltip、页脚、附录、隐藏 sheet、embedded JSON

所以这个 skill 的目标不是“把字段遮住”，而是：

1. 让真实身份信息不外泄
2. 让真实经营信息不外泄
3. 让**所有数字不原样保留**
4. 让脱敏后的材料依然能讲清楚趋势、结构和结论
5. 避免别人通过多个页面、分子分母或图表细节把真实值反推回来

## 它和普通脱敏工具有什么不同

### 1. 文件型优先

这个 skill 把文件型交付物放在第一优先级，而不是只处理文本：

- Excel / CSV
- PPT
- Word
- HTML / Markdown

因为真实世界里，最容易泄漏数据的不是一段句子，而是附件、附表、图表、隐藏 sheet、页脚、注释和 embedded 数据。

### 2. 兼容内容型脱敏

如果你没有上传文件，只是贴了一段内容，它也能工作：

- 一段复盘文字
- 一张表格片段
- 一段 SQL 查询结果
- 一页 PPT 的标题和 bullet
- 一段分析结论

### 3. 不只处理字段，也处理表达

这个 skill 不只做：

- 手机号打码
- 会员 ID 替换
- 品牌名泛化

还会同时考虑：

- 图表标签
- tooltip
- 页脚 / 附录
- 文本结论
- 对外分享时的表达方式

### 4. 默认把数字视为敏感

这点是核心。

在这个 skill 里：

**只要是数字，就默认视为敏感。**

不管它是：

- 人数
- 金额
- GMV
- 收入
- ROI
- 客单价
- 订单量
- 转化率
- 复购率
- 排名值

都不能原样保留。

## 轻度脱敏 vs 重度脱敏

### 轻度脱敏

适合：

- 发给当前客户继续讨论
- 外部沟通但仍然需要较强可读性

特点：

- 用户字段掩码或稳定匿名
- 经营字段做泛化替换
- 所有数字做扰动
- 保留趋势、排序、结构和叙事逻辑

### 重度脱敏

适合：

- Excel 明细外发
- 案例分享
- 培训材料
- 跨客户复用
- 公开或半公开材料

特点：

- 用户字段完全泛化
- 经营字段抽象到类目层
- 所有数字不保留精确值
- 用区间、量级、等级、指数或描述性表达替代

## 这个仓库里包含什么

```text
data-desensitization-skill-repo/
├── README.md
├── data-desensitization/
│   ├── SKILL.md
│   └── references/
│       ├── checklist.md
│       ├── field-catalog.md
│       ├── output-guidelines.md
│       └── rule-matrix.md
└── examples/
    ├── data-desensitization-skill-design.html
    └── data-desensitization-examples.html
```

### `data-desensitization/SKILL.md`

主 skill 文件，定义：

- 什么时候触发
- 文件型与内容型的处理边界
- 轻度 / 重度脱敏的判断逻辑
- 用户只说一句“帮我脱敏这个 Excel”时应该怎么响应

### `references/`

这里放的是可复用规则：

- 敏感字段清单
- 规则矩阵
- 交付物处理规则
- 脱敏检查清单

### `examples/`

这里放的是可直接打开的 HTML 示例：

- 为什么这个 skill 不是简单打码
- 脱敏前后示例，支持轻度 / 重度切换

其中最直观的是：

- `examples/data-desensitization-examples.html`
  用交互式方式展示“脱敏前 vs 脱敏后”，并支持切换 `轻度脱敏` 和 `重度脱敏`

- `examples/data-desensitization-skill-design.html`
  解释这个 skill 的设计思路、数字为什么默认视为敏感，以及轻重度脱敏的边界

## 典型使用场景

### 场景 1：Excel 明细外发

```text
帮我对这个 Excel 做脱敏，按重度脱敏处理。用户字段全部匿名，所有数字都不能保留原值。
```

### 场景 2：发给当前客户的 PPT

```text
把这份 PPT 做轻度脱敏，发给当前客户继续讨论，保留趋势和结构，但真实数字全部扰动。
```

### 场景 3：SQL 查询结果复用

```text
把这段 SQL 查询结果脱敏，去掉可识别字段，并把所有数字改成不可反推的表达。
```

### 场景 4：案例分享版复盘

```text
把这个项目复盘改成案例分享版，按重度脱敏，只保留方法论和启发，不保留真实经营值。
```

## 交互式示例

如果你想快速理解这个 skill 怎么工作，建议先看这个文件：

`examples/data-desensitization-examples.html`

它展示了几类典型内容的脱敏前后对比：

- 用户明细表
- 活动结果页
- 门店差异页
- 字段级规则对照

并且支持切换：

- `轻度脱敏`
- `重度脱敏`

这比只看规则文档更直观，特别适合第一次接触这个 skill 的人。

## 这个项目当前的定位

这是一个 **可安装、可复用、文件型优先、兼容内容型** 的通用数据脱敏 skill v1。

它已经适合直接使用。后续如果继续沉淀，可以增强这些方向：

- 行业化字段清单
- 更稳定的数字扰动策略
- Excel / PPT / HTML 的高风险位置样例
- 面向零售、餐饮等行业场景的默认表达模板
