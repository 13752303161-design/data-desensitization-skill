内部 Skill 设计说明 · 面向对外客户复盘材料

数据脱敏 Skill 应该怎么设计

这不是一个“把手机号打星号”的小工具，而是一套面向 对外发客户复盘材料 的安全改写规则。 你的关键要求已经很清楚：用户字段要脱敏，并且 所有数字都不能原样保留，必须经过干扰或替换。 所以这个 skill 的本质，不是“字段隐藏”，而是 隐私脱敏 + 数字扰动 + 对外展示重写。

为什么你需要单独做这个 skill

原因 1：你的工作天然高风险
你经常要把分析结果转成 PPT、Word、Excel、案例沉淀页。这些交付物一旦包含真实手机号、会员 ID、门店经营数字、活动真实产出，就存在被二次传播的风险。
原因 2：普通“打码”不够
很多人以为脱敏只是把手机号改成 138****1234。但你的场景里，人数、GMV、转化率、门店业绩、券核销量这些数字，同样会暴露客户真实经营情况。
原因 3：你需要稳定规则，不是临场判断
每次都靠你自己临场想“这个数字要不要改”“这个字段能不能露”，既慢又不稳定。skill 的意义，就是把这件事标准化。

这次先锁定的设计边界

1
主场景
对外发客户复盘材料
2
脱敏强度
轻度 / 重度 两档
3
重点对象
用户字段 + 所有数字
4
默认目标
无法直接还原真实数据
最重要的设计原则
对外材料中的真实数字默认不能保留。哪怕是转化率、客单价、人数、GMV，也必须经过统一扰动、区间化、指数化或描述性替换，避免被反推出真实经营数据。

第一层：识别什么要脱敏

字段识别层 敏感对象分类
用户隐私字段手机号、姓名、会员 ID、地址、身份证号、邮箱、设备 ID、订单号等。对外材料里，这类字段默认不能直接出现。
经营敏感字段门店编码、内部渠道编码、商品内部代号、客户内部策略标签、运营脚本命名、私域标签体系等。这次虽然你主要强调用户字段，但经营敏感字段后续也建议纳入规则。
数字敏感字段人数、单量、GMV、核销数、参与数、客单价、转化率、复购率、门店业绩、活动 ROI 等。这里的重点是：这些数字不允许原样保留。

第二层：决定怎么脱敏

脱敏策略层 同字段不同强度
文字型字段可以用掩码、稳定匿名、泛化替代。比如手机号打星，姓名改成“用户 A”，门店名改成“华东门店组 1”。
数字型字段轻度脱敏时做扰动缩放；重度脱敏时改成区间、指数、等级、趋势描述。关键不是“看上去像变了”，而是不能被轻易反推。
图表与结论图表里的柱高、折线、标签、注释、结论描述要保持一致。不能表格里改了数字，PPT 页脚还留着真实值。

第三层：按输出场景套默认规则

输出场景层 交付物默认策略
PPT / Word 对外汇报默认轻度脱敏，但所有数字仍需扰动。目标是保留业务结构和可讲述性，同时去掉真实值。
案例沉淀 / 培训分享默认重度脱敏。重点保留方法论和结构，不保留真实经营数字。
Excel 附表 / 明细导出默认按重度处理，尤其是用户级明细。通常不建议对外发真实粒度明细，哪怕已经打码，也容易联动还原。
轻度脱敏 vs 重度脱敏

轻度脱敏

定位
适合发给当前客户、合作方、外部沟通材料，但不希望真实数据被二次扩散。
目标
保留业务结构、相对关系和可读性，让材料还能正常讲述。
用户字段
用掩码或稳定匿名，比如 138****1234、用户A、M_00128。
数字字段
全部做扰动，不能保留真实绝对值。一般保留趋势、排序、增减关系。
适合场景
客户汇报、项目复盘、合作沟通材料。

重度脱敏

定位
适合案例外发、跨客户分享、培训、方法论宣传。
目标
尽量只保留结构和启发，不保留任何可回推真实业务值的数字与明细。
用户字段
改成完全匿名的角色化描述，如 用户分组1、高活跃用户组。
数字字段
改成区间、等级、指数、相对变化或趋势语言，例如“接近20%”“1万+”“中高水平”。
适合场景
培训分享、案例库、跨品牌内部复用材料。
数字怎么脱敏：不是遮掉，而是统一扰动

你的要求已经明确：数字不能保留原值。所以这个 skill 需要内置“数字扰动规则”，并且保证同一份材料内逻辑一致。

数字类型 轻度脱敏 重度脱敏 设计原则
人数 / 单量 / 核销数 统一比例缩放或区间扰动，比如 12,486 → 13,100 1万+、高位区间、低万级 保留数量级和相对关系，但去掉真实值
GMV / 销售额 / 消费金额 按固定扰动规则改写，如 286,420 → 301,000 中高水平、30万级附近、指数化值 不能通过多页交叉还原真实金额
转化率 / 复购率 / 打开率 全部微扰，如 18.7% → 17.9% 接近20%、高于基线、中位偏上 既不能原值保留，也不能让分子分母反推出来
客单价 / 单店产出 扰动后保留档位感 70元档、中高客单、低客单型 对外更多强调结构，不强调精确值
环比 / 同比变化 可保留方向，但幅度建议扰动 改成“明显提升 / 小幅波动 / 明显回落” 避免通过变化率联动推回原值
数字处理的底线
同一份材料里，如果第一页改了 GMV，后面的图表、注释、页脚、案例引用都必须跟着改。否则哪怕每页单独看都“脱敏”了，跨页合起来仍然能还原真实数据。

这个 skill 未来的工作流

第 1 步：判断交付物类型
PPT、Word、Excel、案例页，不同交付物默认用不同强度。
第 2 步：识别敏感字段
先扫用户字段、经营字段、数字字段，再决定哪些必须处理。
第 3 步：选择脱敏等级
轻度用于客户当前沟通，重度用于案例外发或培训。
第 4 步：统一改写
文本、图表、表格、页脚、结论口径一起改，避免局部脱敏。
第 5 步：输出脱敏说明
告诉你哪些字段被处理了、用了哪种规则，避免黑箱。

为什么我建议先做“规则 + 模板型 skill”

原因 1：你现在最需要标准，不是全自动先把判断规则和默认动作固定住，比上来就写复杂脚本更重要。
原因 2：你的材料类型很多PPT、Word、Excel、案例页都不一样，先把策略讲清楚，后面再考虑脚本自动化。
原因 3：规则型更容易沉淀经验你做过几轮对外复盘之后，就能把真实经验继续补进 skill 里，比如哪些字段最容易漏、哪些数字最敏感。

skill 名称建议

后续最值得沉淀的真实内容

你最常见的敏感字段
常见导表里哪些字段经常出现、哪些字段最容易漏脱敏。
数字扰动默认策略
比如人数、金额、转化率分别怎么改，是否用统一比例、区间、指数。
对外汇报的默认规则
哪些页能保留趋势、哪些页只能保留结构、哪些明细表默认不出。
典型案例模板
比如“鱼你在一起对外复盘版”“零售品牌活动案例分享版”。

我建议的落地顺序

第一步

先写规则版 skill把轻度/重度、字段分类、数字扰动和交付物默认规则先固化。

第二步

补真实案例从你最常见的对外客户复盘材料出发，补真实字段、真实口径、真实坑点。

第三步

再考虑自动化脚本等规则稳定了，再补 CSV / Excel / 文本自动脱敏脚本，避免第一版过重。
