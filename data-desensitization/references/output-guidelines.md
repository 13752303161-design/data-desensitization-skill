# 交付物处理规则

## Excel / CSV

默认规则：
- 对外 Excel / CSV -> 默认重度脱敏
- 如果用户只说 `Excel 脱敏`，默认走重度
- 只有明确说明“发给当前客户继续沟通且要保留较强可读性”时，才考虑轻度

重点检查：
- 原始 sheet
- 隐藏 sheet
- 公式
- pivot
- 图表数据源
- 备注列

## Word

默认规则：
- 发给当前客户的报告 -> 轻度脱敏
- 案例复用 / 外发总结 -> 重度脱敏

重点检查：
- 正文数字
- 表格
- callout
- 页眉页脚
- 图片说明
- 附录

## PPT

默认规则：
- 发给当前客户的对外汇报 -> 轻度脱敏
- 案例分享 deck -> 重度脱敏

重点检查：
- 标题 summary 数字
- 图表标签
- 图表 tooltip
- 页脚
- 附录页
- 备注说明

## HTML / Markdown

默认规则：
- 对外 HTML / Markdown -> 按受众判断轻度或重度
- 如果可公开访问或会复用，优先重度

重点检查：
- 可见文本
- 表格值
- embedded JSON
- tooltip
- hidden fields
- comments

## 内容型输入

默认规则：
- 如果只是让 AI 帮你把一段内容脱敏，默认先按轻度处理
- 如果用户说后续要外发、培训、复用，改成重度

重点检查：
- 品牌 / 门店 / 用户标识
- 所有数字
- 结论表达
- 是否还能被反推
