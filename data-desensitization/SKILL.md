# Data Desensitization

## 概述

这个 skill 是一个 **通用数据脱敏 skill**。

它不是只服务“对外复盘报告”，也不是只做“手机号打码”。
它面向更广泛的数据脱敏场景，优先处理文件型交付物，同时兼容内容型输入：

- 文件型：
  - Excel / CSV
  - Word
  - PPT
  - HTML / Markdown
- 内容型：
  - 文本段落
  - 表格片段
  - SQL 查询结果
  - 图表说明文字
  - 截图中的结构化信息说明

核心目标是：

1. 真实身份信息不能外泄
2. 真实业务敏感信息不能外泄
3. **所有数字不能原样保留**
4. 脱敏后仍然保留分析结构和业务可读性
5. 不让外部接收方轻易反推出真实数据

## 什么时候使用

当用户出现以下需求时触发：

- 明确说 `脱敏`、`匿名化`、`打码`、`anonymize`
- 说要处理 Excel / CSV / PPT / Word / HTML 中的敏感数据
- 说要把 SQL 查询结果、表格片段、文案内容做脱敏
- 说要对外发送、分享、培训、案例沉淀、跨项目复用
- 说要替换品牌名、门店名、策略名、用户标识
- 说数字不能保留原值

如果用户只是做内部原始分析，不要默认触发，除非用户明确要求脱敏。

## 适用范围

### 文件型优先

默认优先处理这些文件：

- Excel / CSV
- Word
- PPT
- HTML / Markdown

原因：
- 文件最容易对外扩散
- 文件里常有隐藏 sheet、公式、图表标签、页脚、注释等高风险内容
- 文件脱敏对一致性要求最高

### 内容型兼容

如果用户没有给文件，而是直接贴出内容，也要能工作：

- 一段复盘结论
- 一段表格
- 一段 SQL 结果
- 一页 PPT 的标题和 bullet
- 一段报告正文

这种情况下，按“内容脱敏”方式处理，但仍然遵守同样的字段和数字规则。

## 核心原则

1. 真实用户标识不能保留
2. 真实经营敏感标识不能保留
3. **真实数字不能原样保留**
4. 图表、表格、正文、页脚、tooltip、注释要一致处理
5. 允许保留业务结构，不允许保留真实值
6. 脱敏后的内容不能轻易被交叉反推

## 脱敏等级

### 轻度脱敏

适用于：
- 发给当前接收方继续沟通
- 外部沟通但仍需较强可读性
- 需要保留趋势、结构、排序、分析逻辑

规则：
- 用户字段做掩码或稳定匿名
- 经营敏感名称做泛化替换
- **所有数字都做扰动**
- 保留高低、趋势、结构、排序
- 文本仍然可以保留业务分析表达

### 重度脱敏

适用于：
- Excel 明细外发
- 案例分享
- 培训复用
- 跨项目传播
- 对外公开或半公开材料

规则：
- 用户字段做完全泛化
- 经营字段做类目级抽象
- **所有数字都不能保留精确值**
- 数字改成区间、量级、等级、指数或描述性表达
- 重点保留方法论和结构，不保留真实经营信息

## 怎么判断用轻度还是重度

按以下优先级判断：

1. 用户明确指定 `轻度脱敏` 或 `重度脱敏` 时，以用户指定为准
2. 用户未指定时，按输入类型和用途判断默认值
3. 如果用途不清楚且风险较高，补一问确认

### 默认判断规则

- `Excel / CSV` 外发或明细导出：默认 **重度脱敏**
- 发给当前接收方继续讨论的 `PPT / Word / HTML 报告`：默认 **轻度脱敏**
- 案例分享 / 培训 / 跨项目复用 / 公开展示：默认 **重度脱敏**
- 用户只说 `帮我脱敏这个Excel`：默认 **重度脱敏**
- 用户只贴一段文本，没有说明用途：默认先按 **轻度脱敏** 处理，并提示可切到重度

## 默认回应方式

如果用户没有指定等级，不要默默随机选择。应明确告诉用户默认采用哪一档，以及为什么。

### 示例 1：用户说

`我要对这个Excel进行脱敏`

推荐回应：

`可以。我会先按重度脱敏处理这份 Excel，因为 Excel 明细对外风险最高。默认规则是：用户字段匿名化，所有数字不保留原值，统一改成扰动值、区间值或等级表达；隐藏 sheet、注释、图表标签和公式引用也一起检查。如果这份文件其实只是发给当前接收方继续沟通，我也可以改成轻度脱敏。`

### 示例 2：用户说

`我要把这份PPT发给当前接收方，先帮我脱敏`

推荐回应：

`可以。我会先按轻度脱敏处理这份 PPT：保留趋势、结构和可讲述性，但所有真实数字都会做扰动，品牌名、用户字段和敏感标识也会一起处理。`

### 示例 3：用户说

`我要把这个项目做成案例分享，帮我脱敏`

推荐回应：

`可以。我会按重度脱敏处理这份案例材料：用户字段完全匿名化，所有数字不保留原值，改成区间、等级、量级或描述性表达，只保留方法论结构和核心启发。`

### 示例 4：用户只贴一段内容

`帮我把这段复盘文字脱敏`

推荐回应：

`可以。我先按轻度脱敏处理这段内容：去掉品牌/用户标识，并把所有数字改成扰动值或泛化表达。如果这段文字后续要做案例外发，我可以进一步改成重度脱敏版本。`

## 敏感对象分类

默认检查四类：

1. 用户隐私字段
2. 经营敏感字段
3. 数字敏感字段
4. 文本叙事敏感内容

详细字段见 `references/field-catalog.md`。

## 数字处理规则

这是本 skill 最重要的规则之一：

**只要是数字，就默认视为敏感。**

包括但不限于：

- 人数
- 金额
- GMV
- 收入
- ROI
- 客单价
- 订单量
- 券成本
- 转化率
- 复购率
- 单店产出
- 比例拆分结果
- 图表中的显示数值

### 轻度脱敏

- 所有数字都做扰动
- 保留趋势、高低、排序、方向
- 数值表达仍然接近业务语言

### 重度脱敏

- 所有数字都不保留精确值
- 改成区间、量级、等级、描述性表达、指数化结果
- 避免分子分母联动反推真实值

### 硬约束

- 不要在整份材料中使用固定公开比例
- 不要在最终输出中暴露扰动规则
- 不要让表格、图表、正文之间相互还原真实值

详细规则见 `references/rule-matrix.md`。

## 文件型处理要点

### Excel / CSV

- 检查原始 sheet
- 检查隐藏 sheet
- 检查公式引用
- 检查 pivot / chart source
- 检查导出时的明细粒度

### Word

- 检查正文数字
- 检查表格
- 检查 callout
- 检查页眉页脚
- 检查图片说明和附录

### PPT

- 检查标题 summary 数字
- 检查图表数据标签
- 检查页脚和附录
- 检查说明文字

### HTML / Markdown

- 检查 visible text
- 检查 table values
- 检查 embedded JSON
- 检查 tooltip / hidden fields / comments

## 内容型处理要点

如果用户贴的是内容而不是文件：

- 先识别内容类型：文本 / 表格 / SQL结果 / 图表说明
- 再识别敏感字段和数字
- 再按轻度或重度改写

内容型脱敏的重点不是文件结构，而是：

- 保留分析逻辑
- 改掉所有身份标识
- 改掉所有数字原值
- 避免保留可恢复的表达

## 执行流程

1. 判断输入是文件型还是内容型
2. 判断交付物或内容用途
3. 判断脱敏等级
4. 扫描敏感字段和所有数字
5. 采用一致映射和一致数字规则
6. 同步改写表格、图表、正文、脚注、tooltip、附录
7. 做最终验证
8. 视情况附一行脱敏说明

## 多数据源一致性

如果任务涉及多页、多表、多文件或多个附件：

- 映射要一致
- 数字处理强度要一致
- 同一指标要一致
- 不允许跨页组合还原真实值

## 交付物规则

查看 `references/output-guidelines.md`：

- Excel / CSV
- Word
- PPT
- HTML / Markdown
- 内容型输入

## 最终验证

在说“已脱敏完成”之前：

- 逐项过 `references/checklist.md`
- 检查图表 tooltip、隐藏字段、备注、附录
- 检查是否还有任何未处理数字
- 检查是否还能交叉反推真实值

## 常见错误

- 只改身份字段，不改数字
- 只改表格，不改图表和文案
- Excel 明细默认做轻度脱敏
- 用固定比例导致容易反推
- 漏掉隐藏 sheet、tooltip、注释、附录
- 认为“文本内容不算数据”，结果把真实数字保留在结论里
