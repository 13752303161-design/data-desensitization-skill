Data Desensitization Skill

一个面向文件型与内容型场景的通用数据脱敏 skill。它不只是“打码”，而是把 敏感字段识别、全部数字扰动、对外表达重写、轻重度规则 组合成一套可复用的方法。

打开交互式脱敏前后示例 查看设计说明 阅读 README

这个 skill 解决什么问题

真实世界的数据脱敏，危险点往往不只是手机号和姓名，还包括品牌标识、门店标识、 用户 ID、订单号、GMV、ROI、转化率、图表标签、隐藏 sheet、页脚和 embedded 数据。 这个 skill 的目标，是让你在保留结构和分析可读性的前提下，避免真实数据被直接看到或被反推。

  • 文件型优先：Excel、CSV、PPT、Word、HTML、Markdown
  • 内容型兼容：SQL 查询结果、表格片段、复盘文字、图表说明
  • 默认把所有数字视为敏感信息，不允许原样保留
  • 支持轻度脱敏和重度脱敏两档规则
2 脱敏强度：轻度 / 重度
4 核心对象：身份字段 / 经营字段 / 数字字段 / 文本叙事
1 底线：真实数字不能原样保留
∞ 可复用场景：报告、案例、附件、分享页、文本片段

从哪里开始看

交互式示例 直接看“脱敏前 vs 脱敏后”的效果，支持切换轻度脱敏和重度脱敏。 设计说明 理解为什么这个 skill 不只是打码，而是包含数字扰动、规则分层和输出重写。 SKILL.md 查看触发条件、默认判断规则、输出方式和对话示例。
如果你是第一次看这个项目，建议顺序：交互式示例 → 设计说明 → SKILL.md。

适合谁

  • 数据分析师与咨询分析团队
  • CRM / CDP / MA 顾问与策略团队
  • 需要把真实业务材料转成可安全分享版本的人
  • 希望把脱敏方法沉淀成可复用 AI skill 的团队

典型输入

  • 帮我对这个 Excel 做重度脱敏，用户字段全匿名，所有数字不能保留原值。
  • 把这份 PPT 做轻度脱敏，保留趋势和结构，但真实数字全部扰动。
  • 把这段 SQL 查询结果脱敏，去掉可识别字段，并把数值改成不可反推的表达。
